## 1. 配置脚本优化

**状态：** 📋 规划中  
**优先级：** ⭐⭐⭐ 高

### 目标
实现一个准确、可用且高可读性的配置脚本格式，整合窗口选择器与元素选择器。

### 核心需求
- **窗口 + 元素一体化选择语法**
  - 将窗口信息（title/class/pid）与 XPath 表达式整合
  - 生成更短、性能更好的选择器
  
- **高可读性**
  - 脚本格式对人类友好，易于理解和维护
  - 支持注释和结构化数据

- **性能优化**
  - 利用窗口预筛选加速元素定位
  - 避免全局搜索，减少 UIA 调用次数

### 示例格式（待讨论）
```yaml
window:
  title: "微信"
  class: "mmui::MainWindow"
  
element:
  xpath: "//Button[@AutomationId='btnSend']"
  # 或更简洁的语法
  selector: "Button#btnSend"
```

## 3. 多屏支持

**状态：** 📋 规划中  
**优先级：** ⭐⭐ 中

### 目标
API 需支持获取元素在多显示器环境下的精确屏幕坐标。

### 核心需求
- **虚拟屏幕坐标系**
  - Windows 多显示器使用虚拟屏幕坐标（主屏左上角为原点）
  - 支持负坐标（扩展屏在主屏左侧/上方）

- **显示器信息**
  - 返回元素所在的显示器编号/名称
  - 支持按显示器过滤窗口列表

- **坐标转换工具**
  - 虚拟坐标 ↔ 显示器相对坐标
  - DPI 感知支持（不同显示器可能有不同缩放比例）

### API 扩展
```json
{
  "rect": {
    "x": 1920,
    "y": 540,
    "width": 100,
    "height": 30,
    "monitor": 2,
    "monitor_name": "\\\\.\\DISPLAY2"
  }
}
```

## 4. 拟人化鼠标模拟

**状态：** 📋 规划中  
**优先级：** ⭐⭐ 中

### 目标
实现更自然的鼠标行为，模拟真实用户操作，避免被反自动化检测。

### 核心功能

#### 4.1 随机移动轨迹
- **贝塞尔曲线/样条插值**
  - 生成平滑的曲线轨迹
  - 避免直线移动（明显是机器人）

- **随机扰动**
  - 在轨迹上添加微小的随机偏移
  - 模拟人手抖动

#### 4.2 智能定位
- **目标区域随机选择**
  - 点击位置在元素矩形区域内的 **50%-60% 范围**内随机选取
  - 避免总是点击元素中心点

- **边界安全**
  - 确保点击位置不会超出元素边界
  - 考虑元素的可点击区域（排除 padding 等）

#### 4.3 行为模拟
- **加速/减速曲线**
  - 启动时加速，到达目标前减速
  - 符合 Fitts's Law 人体工学模型

- **点击后空闲移动**
  - 点击完成后，继续模拟随机的鼠标空闲移动
  - 随机停顿时间（200ms - 2000ms）

- **滚动模拟**
  - 支持鼠标滚轮操作
  - 随机滚动速度和距离

### API 设计（示例）
```python
# Python SDK 示例
mouse.click_element(
    window={"title": "微信"},
    xpath="//Button[@AutomationId='btnSend']",
    humanize=True,           # 启用拟人化
    random_range=0.55,       # 50%-60% 范围
    trajectory="bezier",     # 贝塞尔曲线轨迹
    pause_after=1000         # 点击后停顿 1 秒
)
```

### 技术实现
- **Windows API**
  - `SendInput` - 模拟鼠标输入
  - `mouse_event` - 传统方式（兼容旧系统）

- **轨迹算法**
  - 三次贝塞尔曲线
  - Perlin 噪声（随机扰动）
