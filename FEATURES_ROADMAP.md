## 3. 多屏支持

**状态：** 📋 规划中  
**优先级：** ⭐⭐ 中

### 目标
API 需支持获取元素在多显示器环境下的精确屏幕坐标。

### 核心需求
- **虚拟屏幕坐标系**
  - Windows 多显示器使用虚拟屏幕坐标（主屏左上角为原点）
  - 支持负坐标（扩展屏在主屏左侧/上方）

- **显示器信息**
  - 返回元素所在的显示器编号/名称
  - 支持按显示器过滤窗口列表

- **坐标转换工具**
  - 虚拟坐标 ↔ 显示器相对坐标
  - DPI 感知支持（不同显示器可能有不同缩放比例）

### API 扩展
```json
{
  "rect": {
    "x": 1920,
    "y": 540,
    "width": 100,
    "height": 30,
    "monitor": 2,
    "monitor_name": "\\\\.\\DISPLAY2"
  }
}
```

## 4. 拟人化鼠标模拟

**状态：** 🔄 设计中（与 HTTP 服务同步开发）  
**优先级：** ⭐⭐⭐ 高

### 目标
实现更自然的鼠标行为，模拟真实用户操作，避免被反自动化检测。

### 核心功能

#### 4.1 随机移动轨迹
- **贝塞尔曲线轨迹**
  - 三次贝塞尔曲线生成平滑轨迹
  - 控制点随机扰动，模拟人手抖动

- **缓动函数**
  - 启动时加速，到达目标前减速（ease-in-out）
  - 符合 Fitts's Law 人体工学模型

#### 4.2 智能定位
- **目标区域随机选择**
  - 点击位置在元素矩形区域内的 **50%-60% 范围**内随机选取
  - 避免总是点击元素中心点

- **边界安全**
  - 确保点击位置不超出元素边界

#### 4.3 行为模拟
- **点击停顿**
  - 点击前/后随机停顿时间（可配置）

### 坐标计算规则
```typescript
// 中心坐标
center = { x: rect.x + rect.width / 2, y: rect.y + rect.height / 2 }

// 随机坐标（50%-60% 范围）
// rangePercent = 0.55 表示在中心 55% 范围内随机
centerRandom = {
  x: center.x + random(-width * 0.55 / 2, width * 0.55 / 2),
  y: center.y + random(-height * 0.55 / 2, height * 0.55 / 2)
}
```

### 技术实现
- `SendInput` - Windows 鼠标输入模拟
- 三次贝塞尔曲线算法
